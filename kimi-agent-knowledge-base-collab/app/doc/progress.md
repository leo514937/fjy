完成本体知识实验室的最后打磨，清理开发痕迹，优化浅色模式视觉表现，并修复 UI 细节缺陷。

## 已完成任务
2. **侧栏与工作台 UI 极致打磨**：
   - **侧栏布局优化**：通过 `justify-end` 实现控制台内容整体下压，并保持块级紧凑，最大限度减少底部冗余空白。
   - **元素尺寸加厚**：增大了导航标签、搜索框、筛选按钮及统计卡片的尺寸与内边距，提升工业感与易用性。
   - **导航顺序调整**：优化侧边栏 Tab 顺序为：本体库 -> 问答助手 -> 本体图谱 -> 小故Git。
3. **图谱工作台交互优化**：
   - **按钮效果增强**：提升了“详细参数”与“实体选取”按钮的视觉对比度与激活反馈。
   - **滚动区域分离**：为详情与选取面板配置了独立的 `ScrollArea`，实现滚动状态的物理分离，互不干扰。
4. **代码逻辑精简与修复**：
   - **执行流优化**：移除了“执行流程”中的时间轴连接长线，视觉更清爽。
   - **告警清理**：彻底清理了 `ExecutionFlow.tsx` 中的冗余导入、变量及 Props，并同步修正了 `OntologyAssistant.tsx` 中的类型调用错误。
5. **开发痕迹清理 (交付标准)**：
   - 彻底移除了 WiKiMG 源码中 (`遥测字段规范.md`) 的“前端联调提示” Callout。
   - 在 `MarkdownBlocks.tsx` 渲染层增加了过滤机制，确保带有“前端联调提示”标题的块永远不会被展示。
2. **代码质量修复**：
   - 解决了 `BrowsePage.tsx` 使用 `OntologyBrowser` 组件时的属性不匹配导致的 TS 飘红问题（增加了 `onSelectEntity` 和 `selectedEntityId` 支持）。
3. **视觉一致性优化**：
   - **代码块样式升级**：针对浅色模式，将代码块背景从深黑色调整为工业灰 (`bg-zinc-50`)，并优化了 Language Tag 的 Header 样式，提升了整体视觉的和谐度。
4. **布局缺陷修复**：
   - **内容遮挡修复**：在 `EntityDetail.tsx` 的底部增加了足够的留白和 Padding，解决了在部分高度受限容器中底部元信息（NODE ID 等）被截断的问题。
