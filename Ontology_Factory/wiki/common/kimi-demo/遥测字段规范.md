# 遥测字段规范

> 一套共享的命名和单位约定，用来把不同设备的上报数据放到同一种语言里。

## 定义与定位
遥测字段规范属于 `common` 层的共享规范节点。它不直接代表某个具体设备，而是为平台接入、监测面板和设备配置提供统一的数据表达方式。

## 属性
- 核心目标: 统一字段名、单位、时间粒度
- 典型字段: 温度、水质、溶氧、告警状态
- 使用对象: 平台接入工程、监测面板、运维排障

## 字段模板
| 字段 | 含义 | 示例 |
| --- | --- | --- |
| `device_id` | 设备编号 | `sensor-oxygen-01` |
| `metric_key` | 指标键名 | `dissolved_oxygen` |
| `captured_at` | 采集时间 | `2026-04-11T09:30:00+08:00` |

```json
{
  "device_id": "sensor-oxygen-01",
  "metric_key": "dissolved_oxygen",
  "metric_value": 6.2,
  "unit": "mg/L"
}
```

> [!NOTE] 前端联调提示
> 如果页面要做趋势图、卡片和告警联动，字段命名和单位必须保持稳定。

## 证据来源
- 多设备协同时，字段名和单位不统一会直接影响聚合展示。
- 平台接入和面板渲染都依赖统一的数据结构。
