🛠️ 小故Git (XiaoGuGit)
—— 专为智能体设计的工程文档版本管理中间件
小故Git 是一个基于 FastAPI 和 Git 的轻量级持久化服务。它将复杂的 Git 版本管理指令封装为简单的 RESTful API，专门用于存储 AI 智能体生成的结构化数据（如本体 JSON、功能模块分解图等），并提供完整的审计追踪和版本回溯能力。

🌟 核心特性
语义化版本记录：不同于传统 Git 需人工编写提交信息，小故Git 接收智能体生成的 message，让每一条版本记录都具备业务意义。

安全回滚 (Incremental Revert)：采用非破坏性的 revert 机制。回滚操作不会抹除历史，而是通过产生一个“补偿版本”来恢复数据，确保证据链完整。

多项目物理隔离：每个 project_id 拥有独立的 Git 仓库目录，互不干扰。

行级差异对比：提供标准 Git Diff 输出，为 LLM 生成抽象规约或变更审计提供精准素材。

环境兼容性：完美运行于 Anaconda 等 Python 环境，支持中文 UTF-8 无损存储。

📑 接口规范 (API Reference)
1. 智能写入版本 (POST /write)
智能体完成提取或分解任务后调用，将数据存入“保险箱”。

请求体 (JSON)：

JSON
{
  "project_id": "engine_p1",
  "filename": "ontology.json",
  "data": { "nodes": ["发动机", "燃烧室"] },
  "message": "AI: 细化了燃烧室节点分解",
  "agent_name": "Decomposer_01"
}
响应：返回 commit_id (40位哈希值)。

2. 语义化读取 (GET /read)
获取文件内容。支持通过 commit_id 穿梭到任意历史时刻。

路径参数：project_id, filename

查询参数：commit_id (可选，不传则读最新版)

3. 版本日志查询 (GET /log)
检索项目的所有历史演化记录。

返回：包含 id, agent, msg (AI 摘要), time 的列表。

4. 差异对比 (GET /diff)
获取两个版本间的物理差异文本。

查询参数：base(旧版本ID), target(新版本ID)

5. 安全回滚 (POST /rollback)
将文档内容恢复到指定版本。

机制说明：不同于 reset --hard，该接口使用 revert 逻辑，会产生一个新的提交来记录这次“恢复”行为，保证版本树递增。

🛡️ 安全回滚机制详解
在 manager.py 中，回滚逻辑已升级为安全模式：

Python
def rollback(self, project_id, commit_id):
    repo = self._get_repo(project_id)
    # 撤销改动而不抹除历史
    repo.git.revert(f"{commit_id}..HEAD", n=True)
    author = git.Actor("System", "system@local")
    # 产生新版本记录回滚操作
    repo.index.commit(f"System: 自动回滚至版本 {commit_id}", author=author)
    return True
这意味着： 如果你回滚错了，你甚至可以再次回滚回去，因为没有任何数据会被真正删除。

⚠️ 注意事项与建议
路径安全：当前版本未对 project_id 做严格路径校验，请确保调用方可信，避免路径穿越。

并发控制：Git 仓库在执行写操作时会有文件锁，建议避免多个智能体在同一秒内高频并发写入同一项目。

数据格式：推荐统一使用 .json 后缀，以便 Git 实现最优的行级差异对比。
