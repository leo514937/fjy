# 鱼家——智能养鱼系统 CDIO 项目流程
## 一、Conceive（构想）
### （一）项目目标
由生活出发，我们组想要实现远距离网络测量并调节鱼缸的温度，充氧，饲料的添加，改变鱼缸环境的光线，给播放舒缓的音乐等。
冯韵菱第一次注意到宿舍角落的鱼缸，是在 2018 年 11 月初的一个周末。那几天期中考试临近，大家连吃饭都常常忘记，更别说照顾鱼了。鱼缸里的一条小红斑鱼在水面附近急促换气，像是在无声地求救。温度下降、光照不足，再加上朋友忘记投喂，鱼缸的环境正在悄悄恶化，而宿舍里没有人能意识到这些变化。

“如果我们可以在手机上看到它们的状态就好了。”
那天晚上，在活动中心的桌子旁，五个人开始第一次讨论这个想法。他们发现，身边很多同学都有类似的经历：
有人买了热带鱼，却因为周末回家两天，回来时发现几条鱼已经翻肚；
有人实验室的鱼缸周末无人照看，温度骤降导致一缸实验鱼全部死亡；
还有人虽然很喜欢养鱼，但因为忙于课程或社团，经常忘记喂食或开灯。
这些看似小事，却不断重复发生。于是，一个朴素但强烈的需求逐渐被所有人感受到：
能不能让鱼缸“自己照顾自己”？至少在我们忙不过来时，它能保持最基本的稳定和安全。

小组成员开始回想更具象的情境：
冯昕澳说，他爸妈春节外出三天内，家里的金鱼因为连续低温冻死了；
刘晃伸则分享，他同学养的斗鱼曾因为夜里氧泵停了一个小时，第二天就没了；
张靖鸿回忆，自己刚入坑养鱼时，总是搞不清楚到底该投多少食、什么时候该开灯。
这些并非遥远的案例，而是“真实且反复发生”的需求场景。这些故事让他们意识到，这不是单纯的课程作业，而是一个真正能改善日常生活的小系统。

就在 11 月 10 日购买完硬件的那天夜里，小组成员重新明确了项目愿景：
做一个让即使三天不在宿舍、手机也打不开鱼缸、甚至忘了鱼的存在，鱼也能安然活着的系统。

现实中，养鱼爱好者在长时间外出或工作繁忙时，经常出现以下痛点：
水质无人监控（pH/TDS 波动、温度骤变、硝酸盐积累）；
投喂不规律（过量/不足，经常导致鱼类应激或死亡）；
增氧不足或夜间缺氧（如氧泵停止、溶氧下降）；
新手缺乏经验，不懂判断水质/设备异常；
多个设备难以统一控制（加热棒、增氧泵、灯光、投喂器分散）。
随着 ESP8266、Onenet 平台、Arduino、传感器模组的成熟，本项目提供了可行的智能养殖方案。

2. 产品目标（V1）

基础生存保障：远程稳定完成投喂、增氧、温控，保障无人值守期间鱼只存活。
环境透明化：采集并远程查看温度、光照等数据，后续可扩展更多水质参数。
预警可控：当设备反馈异常（如温度低于阈值、光照骤变），可远程介入操作。
稳定系统与闭环体验：软硬件可协同运行，数据上传与指令下发可靠。

## 二、Design（设计）
### （一）功能设计方案
系统需实现三个核心目标：
（1）基础生存保障功能（温控、光照代理环境、投喂）
（2）远程可观测与可控功能（数据可视化、操作下发、故障提示）
（3）安全可依赖功能（加热棒离水断电保护、控制冗余机制）
1. 硬件端功能设计：温度传感器、光敏传感器数据采集，通过ESP8266上传至Onenet；继电器控制开关、小圣诞树、LED时钟；鱼食添加装置、加热棒离水断电装置设计；彩色LED呼吸灯、蜂鸣器音乐伴随彩灯闪烁功能设计。
2. 软件端功能设计：微信小程序界面及整体框架设计，包含主页、获取信息、控制台、关于我们等模块；小程序调用Onenet数据、跳转页面查看数据、发送数据到Onenet功能设计；项目网站初步设计，包含开关灯控制等交互功能。

###（二）系统总体架构
（1）设备层（Device Layer）
Arduino Uno：核心控制逻辑执行。
传感器：温度（A3）、光照（A0）。
执行器：继电器（控制灯/加热棒等）、舵机（投喂）、LED 灯、蜂鸣器。
ESP8266：负责 WiFi 连接与上/下行通信。
（2）网络层（Connectivity Layer）
HTTP（上传温度/光照至 Onenet）
TCP + EDP 协议（接收 Onenet 下行控制指令）
（3）云平台层（Onenet）
数据流（datastream）：Temperature / Light
控制命令：switch0 / switch1
EDP 指令推送与解析
（4）应用层（Application Layer）
微信小程序：数据展示（图表）、设备控制
Web 页面：简易灯光开关界面

（三）核心功能设计
1. 传感采集功能（Sensor Acquisition）
Arduino 周期采集两项数据：
温度 sensor_tem（来自 A3）
光照 sensor_lux（来自 A0）
1、数据上传机制
1）Arduino 采样
生成 JSON：
{
  "Temperature":"xx.xx",
  "Light":"xx.xx"
}
2）通过 ESP8266 发送 HTTP POST：
POST /devices/{DEVICEID}/datapoints?type=3
api-key:{APIKEY}
Onenet 存储数据流，供小程序读取。
传输间隔：INTERVAL_NET = 10000 ms（代码中明确）

2. 设备控制系统（Actuator Control）
（1）下行控制方式：EDP 协议
【新增：来自解析代码的真实格式】
Onenet 下发指令格式（示例）：
switch0:1
Arduino 使用 sscanf 分解命令，并根据 value 控制继电器端口：
value = 1 → HIGH
value = 0 → LOW
switch1:2 / switch1:3 → 控制 JDQ5（代码中存在）
这是系统唯一可信赖的远程控制链路。

（2）可控制对象
小灯 / 圣诞树灯（JDQ6）
加热棒电源（通过继电器控制）
投喂舵机（由命令触发）

3. 自动投喂功能（Feeding Mechanism）
使用圆柱形料仓开孔 + 舵机旋转。
投喂动作由小程序→Onenet datapoint（switch 类命令）触发
Arduino 根据命令驱动舵机转动一个固定角度
料仓开口对准鱼缸完成投喂

4. 加热棒离水断电机制（Heater Safety）
物理/机械结构实现离水即断电。
此保护不依赖传感器，而是通过机械触发 + 继电器断电实现

5. 可视化与控制界面设计

微信小程序功能（来自原文软件端实现）：
获取数据（温度/光照）
绘制折线图（wxcharts.js）
发送控制命令（开/关灯、投喂等）
页面导航栏（主页、信息、控制台、关于我们）
小程序 → Onenet → ESP8266 → Arduino → 执行器
是完整的控制链路。

bool doCmdOk(String data, char *keyword)
{
  bool result = false;
  if (data != "")    //对于tcp连接命令，直接等待第二次回复
  {
    WIFI_UART.println(data);  //发送AT指令

  {
    delay(2000);
  }
  else{
    while (!WIFI_UART.available());  // 等待模块回复
  }
  
  delay(200);
  
  if (WIFI_UART.find(keyword))    //返回值判断
  {
    DBG_UART.println("do cmd OK");
    result = true;
  }
  else
  {
    
    DBG_UART.println("do cmd ERROR");
    result = false;
  }

  return result;

/*
* readEdpPkt
* 从串口缓存中读数据到接收缓存
*/
bool readEdpPkt(edp_pkt *p)
{
  int tmp;
  if ((tmp = WIFI_UART.readBytes(p->data + p->len, sizeof(p->data))) > 0 )
  {
    rcvDebug(p->data + p->len, tmp);
    p->len += tmp;
  }
  return true;
}

/*
* packetSend
* 将待发数据发送至串口，并释放到动态分配的内存
*/
void packetSend(edp_pkt* pkt)
{
  if (pkt != NULL)
  {
    WIFI_UART.write(pkt->data, pkt->len);    //串口发送
    WIFI_UART.flush();
    free(pkt);                //回收内存
  }
}

void rcvDebug(unsigned char *rcv, int len)
{
  int i;

  DBG_UART.print("rcv len: ");

  for (i = 0; i < len; i++)
  {
    DBG_UART.print(rcv[i], HEX);
    DBG_UART.print(" ");
  }
  DBG_UART.println("");
}
