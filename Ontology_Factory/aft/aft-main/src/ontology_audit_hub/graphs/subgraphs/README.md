# 子图 (Subgraphs) 与 代理 (Agent) 逻辑说明

本目录包含了基于 [LangGraph](https://github.com/langchain-ai/langgraph) 构建的审计工作流核心逻辑。通过将复杂的审计任务拆分为多个子图，实现了高度模块化和可维护性。

## 🤖 调度器代理 (Supervisor Agent)

调度器是整个系统的“大脑”，负责解析用户意图、协调子图运行、整合审计结果，并处理“人在回路”（Human-in-the-loop）的交互。

       [ START ]
           |
    +------v------+
    | load_request| (加载审计请求)
    +------+------+
           |
    +------v-------+
    | intent_router| (意图路由：解析用户想做什么)
    +------+-------+
           |
    +------v----------+
    |route_to_subgraph| (分流器)
    +------+----------+
           |
           +-----------------------+-----------------------+-----------------------+
           |                       |                       |                       |
  +--------v-------+      +--------v-------+      +--------v-------+      +--------v-------+
  | ontology_audit |      | document_review|      |   code_test    |      |   full_audit   |
  |  (本体检查)    |      |  (文档审查)    |      |  (代码测试)    |      |  (全流程审计)  |
  +--------+-------+      +--------+-------+      +--------+-------+      +--------+-------+
           |                       |                       |                       |
           +-----------------------+-----------+-----------+-----------------------+
                                               |

## 🔍 子图流程细节 (逻辑分解)

虽然三个子图在功能上都是**链式 (Sequential)** 的，但实现的颗粒度不同：
- **前两个 (Ontology & Document)**：将所有逻辑封装在 `core` 节点中，适用于逻辑紧凑、不需要多个断点的操作。
- **第三个 (Code)**：将每一个步骤定义为独立的 Graph 节点，便于状态跟踪、断点调试和更精细的错误处理。

### 1. 本体审计 (Ontology Subgraph)
**核心逻辑：读取并验证。**

```text
 [ 读取 YAML ] -> [ 实体校验 ] -> [ 关系一致性检查 ] -> [ 生成本体对象 ]
```

### 2. 文档审查 (Document Subgraph)
**核心逻辑：对比与检索。**

```text
 [ Markdown 分块 ] -> [ Qdrant 语义检索 ] -> [ 提取并比对断言 (Claims) ] -> [ 标识规则冲突 ]
```

### 3. 代码分析 (Code Subgraph)
**核心逻辑：多步静态/动态测试链。**
(此子图将每一步拆分为显式的 Graph 节点)

```text
       [  收集 Python 源文件  ]
                  |
       [  AST 语法树解析/提取 ]
                  |
       [  推断代码与本体的绑定 ]
                  |
       [  与本体定义进行合规性比对 ]
                  |
       [  自动为本体约束生成测试规格 ]
                  |
       [  生成物理 Pytest 测试文件 ]
                  |
       [  执行 Pytest (动态验证) ]
                  |
       [  汇总测试结果至审计发现 ]
```
