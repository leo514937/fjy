# 功能说明文档：本体协商与“达、类、私”架构

本项目的核心目标是构建一个基于逻辑学层级的情报处理系统。其架构逻辑源自中国古代墨家逻辑（《墨子·经上》），通过“达、类、私”三个核心概念，实现对复杂知识图谱的结构化治理。

---

## 1. 核心概念：达、类、私

在系统中，实体的分类不再仅仅是宽泛的标签，而是遵循严格的逻辑外延划分：

### 达 (Universal / 达名)
- **概念**：外延最大，指代最普遍的范畴。
- **作用**：作为本体图谱的“最高类概念”，负责捕获实体的存在性。在系统中，它代表了所有实体的语义基石。
- **示例**：如“事物”、“实体”、“客体”。

### 类 (Category / 类名)
- **概念**：指称具有相同属性的一类事物，外延处于中间层级。
- **作用**：作为系统最主要的分类引擎，负责实体的行业、功能或学科归类。
- **示例**：如“马”、“城市”、“传感器”、“手机”。

### 私 (Private / 私名)
- **概念**：外延最小，仅特指某一个具体个体。
- **作用**：负责实现从“通用”到“唯一”的精准对齐，确保情报映射到特定的实体。
- **示例**：如“苏格拉底”、“北京”、“特定的 iPhone 序列号”。

---

### 1.4 从扁平到分层：知识的演化路径

在实际应用中，原始的知识图谱往往呈现为一种**扁平化、无深度**的网状结构。系统通过以下机制实现向“达、类、私”分层架构的转化：

1. **语义锚定 (Anchor)**：在全图中自动识别那些具有“最高普适性”的核心词汇，将其定义为**达层**，为后续所有分类建立逻辑原点。
2. **关系聚合 (Aggregation)**：通过拓扑分析与语义聚类，将节点中具有共性的属性提取出来，形成各种“类概念”。这些中层支柱构成了**类层**，决定了知识的分类粒度。
3. **差异化定位 (Differentiation)**：针对特定的实体，利用 Agent 间的反复辩论与证据对账，剔除冗余和歧义，使其最终精准地沉淀在**私层**。

这一过程将原本杂乱无章的“平面数据表”升级为了具有逻辑权重和深度感知能力的“三维知识体系”。

---

## 2. Agent 工作流程

系统通过多 Agent 协作完成实体的层级协商，流程如下：

1. **提案阶段 (Proposer)**：Agent 根据输入的原始数据特征，提出实体的分层建议。例如，基于坐标和名称特征，建议将一个点归入“私名：北京”。
2. **反驳阶段 (Critic)**：Agent 寻找逻辑矛盾或证据缺失。当提案的独特性证据不足（例如名称存在重复可能）时，会提出反向论据。
3. **裁决阶段 (Arbiter)**：基于现有的“达、类、私”逻辑约束，评判双方辩论。若结论不收敛，则要求补充更多证据（如属性验证或邻居节点校验）；若收敛，则执行最终的层级判定。
4. **评估阶段 (Evaluator)**：对最终对齐结果给出置信度分析与逻辑一致性审计报告。

## 3. 设计愿景

通过引入墨家逻辑的分层思想，系统不仅能处理结构化数据，还能通过 Agent 的理性辩论，在复杂的、非确定性的语义环境中，找到最符合逻辑的知识位置。
