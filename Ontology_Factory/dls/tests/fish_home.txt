Fish_Home 智能鱼缸系统：工程化本体文档
一、 达层（L3: Universal - 通用公理层）
定义： 不随项目改变的底层逻辑，属于“免验证底层”。

因果律（故）： * 无之必不然： 若无“能量输入”（电源），则“体”（硬件）无法产生“然”（运行状态）。

封闭反馈： 环境参数（输入）经逻辑变换（算法）必产生执行动作（输出），以维持系统稳态。

拓扑公理： * 感知-决策-执行： 任何智能控制系统必须包含传感器（感知）、控制器（决策）与执行器（执行）三个逻辑实体。

通信公理： * 协议对等： 两个实体间的交互必须建立在波特率、帧结构一致的“同异”基础上。

二、 类层（L2: Class - 领域标准层）
定义： 适用于“智能水族”或“物联网感知控制”领域的标准化定义。

1. 物理架构类 (Physical Structure Class)
感知集群标准： 定义水温（热敏抵抗）与光照（光敏电阻）的线性映射关系。

执行机构标准： 定义开关量控制（氧气泵、灯光）与占空比控制（舵机投食）的控制范式。

2. 通信协议类 (Protocol Class)
端-云映射： 定义 Device -> Gateway (ESP8266) -> Cloud (OneNET) 的三级转发标准。

数据模型： 定义 DataPoint（时序遥测数据）与 DeviceCommand（即时控制指令）的标准 JSON 结构。

3. 表现层类 (Representation Class)
交互规范： 定义实时曲线图的“轮询策略”（5s 间隔）与“状态反馈机制”。

三、 私层（L1: Individual - 具体的项目实例）
定义： 指向 fish_home 项目特有的代码实现、配置与版本记录（Git 存储的核心内容）。

1. 节点实例：arduino_uno_controller (核心大脑)
然-体映射： * 体（结构）： Arduino Uno 硬件引脚分配（A0-温度，D9-舵机）。

然（行为）： fish_home_brain.ino 代码逻辑，9600/115200 双串口并发处理。

Git 记录： FIRMWARE_IMPLEMENTATION_PLAN.md 记录了该实例的演进轨迹。

2. 节点实例：esp8266_wifi_module (通信网关)
名实耦验证： * 实： 当前物理连接至“用于wifi模块”的控制线。

名： STATUS.md 中定义的联网状态。

具体协议： 针对 OneNET 平台的 EDP 协议具体实现。

3. 节点实例：onenet_iot_platform (后端中枢)
实例化代码： 基于 FastAPI 的 main.py 和 config.py。

逻辑边界： 限制 API 接口为 /api/v1/data，仅服务于微信小程序客户端。

4. 节点实例：wechat_mini_program_client (交互终端)
私有逻辑： app.js 逻辑中硬编码的 AppID 及特定 UI 布局。

实验记录： test.py 记录了针对该实例进行的联调实验结果。
